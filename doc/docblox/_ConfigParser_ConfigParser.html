Description

The ConfigParser class implements a basic configuration language which provides a structure similar to what’s found in Microsoft Windows INI files. You can use this to write PHP programs which can be customized by end users easily.

DISCLAIMER: Every docblock was shameless copied or at least adapted from Python's configparser documentation page (version 3.0). See http://docs.python.org/dev/library/configparser.html

getBoolean(  $section,  $option ) :

A convenience method which coerces the option in the specified section to a Boolean value. Note that the accepted values for the option are '1', 'yes', 'true', and 'on', which cause this method to return TRUE, and '0', 'no', 'false', and 'off', which cause it to return FALSE.

These string values are checked in a case-insensitive manner. Any other value will cause it to raise ValueException.

read(  $filenames = array ) :

Attempt to read and parse a list of filenames, returning a list of filenames which were successfully parsed. If filenames is a string, it is treated as a single filename. If a file named in filenames cannot be opened, that file will be ignored. This is designed so that you can specify a list of potential configuration file locations (for example, the current directory, the user’s home directory, and some system-wide directory), and all existing configuration files in the list will be read. If none of the named files exist, the ConfigParser instance will contain an empty dataset. An application which requires initial values to be loaded from a file should load the required file or files using read_file() before calling read() for any optional files:

write_(  $filename = NULL,  $space_around_delimiters = true ) :

Write a representation of the configuration to the specified filename. This representation can be parsed by a future read() call.

If $space_around_delimiters is true, delimiters between keys and values are surrounded by spaces.

When $filename is NULL, the last file loaded (if any) is used.
